Selecting Play! 2.x Provider for Play! 2.3.x

There are three ways of selecting this provider.

Using project property.

Add play2.version project property with value starting with 2.3..

This is simplest and recommended way because this property can be used to declare versions of Play! Framework dependencies.

<project>
    ...
    <properties>
        ...
        <play2.version>2.3.10</play2.version>
    </properties>
    ...
</project>

Using plugin configuration parameter.

Add playVersion parameter with value starting with 2.3. to plugin configuration section.

<project>
    ...
    <build>
        ...
        <plugins>
            ...
            <plugin>
                <groupId>com.google.code.play2-maven-plugin</groupId>
                <artifactId>play2-maven-plugin</artifactId>
                <version>1.0.0-beta6-SNAPSHOT</version>
                <extensions>true</extensions>
                <configuration>
                    ...
                    <playVersion>2.3.10</playVersion>
                </configuration>
            </plugin>
            ...
        </plugins>
    </build>
    ...
</project>

Provider as plugin dependency.

Add provider artifact as plugin dependency.

Warning: Only one provider can be added as plugin’s dependency at a time. It will be selected regardless of the value of playVersion plugin configuration parameter or play2.version project property.

<project>
    ...
    <build>
        ...
        <plugins>
            ...
            <plugin>
                <groupId>com.google.code.play2-maven-plugin</groupId>
                <artifactId>play2-maven-plugin</artifactId>
                <version>1.0.0-beta6-SNAPSHOT</version>
                <extensions>true</extensions>
                <dependencies>
                    <dependency>
                        <groupId>com.google.code.play2-maven-plugin</groupId>
                        <artifactId>play2-provider-play23</artifactId>
                        <version>1.0.0-beta6-SNAPSHOT</version>
                    </dependency>
                </dependencies>
            </plugin>
            ...
        </plugins>
    </build>
    ...
</project>
